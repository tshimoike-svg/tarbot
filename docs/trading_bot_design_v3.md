# 自動売買ボット 引き継ぎ設計書 v3

> Claude Code および上位モデルによる精査・実装引き継ぎ用ドキュメント
> 作成日：2026年6月（v3：時間軸を「日中」から「数日〜2週間のスイング」へ転換）
>
> **v2 からの主な変更点**
> - 時間軸を **日中（分足・当日クローズ）→ 日足スイング（数日〜最大2週間保有）** に転換
> - これにより **分足データ・ライブ常時接続の必要を排除**。J-Quants の日足（無料/有料）で
>   **そのままバックテスト可能**になり、Phase 0 の「安く・速く白黒つける」が口座開通前に成立
> - エッジ候補を2本に：**日足スイング平均回帰**（押し目買い/戻り売り）と
>   **日足ブレイクアウト・モメンタム**。両方を同じ枠組みで検証し比較する
> - 真のコストの中心が **スプレッド＋滑り** から **持ち越し金利＋ギャップリスク** へ比重移動
>   （往復コストはエッジ＝数%の値幅に対して相対的に小さくなる）
>
> v1/v2 は `docs/` の履歴（git）に残る。v3 が現行仕様。

## 0. このボットの勝ち筋（v3）

**前提は不変**：個人が GUI アプリ（kabuステーション）経由のAPIで、ミリ秒の板の歪みを
機関・HFT から取りにいっても勝てない。だから**速度では戦わない**。

v2 は「日中の小さな歪みを指値でスプレッドを捕りながら頻回で取る」方針だったが、これは
(a) 分足データが要る（無料では入手困難）、(b) スプレッド・滑りがエッジに対して大きい、
(c) 一日中ライブ接続が要る、という重い制約を抱えていた。

**v3 はエッジの時間軸を数日〜2週間へ広げる**ことでこれらを一掃する：

1. **容量制約ニッチ（不変）**：機関が資金を入れにくい中小型株の歪みを狙う。これは時間軸では
   なく容量の話なので、スイングでもそのまま成立する。
2. **数日スケールの統計的エッジ**：平均回帰（行き過ぎの反転）と、ブレイクアウト後の継続。
   日足 OHLCV で**過去検証できる**。
3. **コストの相対的低下**：狙う値幅が数%になるため、往復コスト（スプレッド＋滑り＋金利）は
   エッジの小さな割合になる。ただし**持ち越し金利とギャップリスクが新たに効く**。

期待値の式（全戦略がこれを満たさなければ稼働しない・不変）：

```
E[1トレード] = 勝率 × 平均利益 − 敗率 × 平均損失 − 往復コスト
往復コスト ≒ スプレッド（実効） + 滑り + 信用コスト（保有日数ぶんの金利・貸株料）
稼働条件：E[1トレード] > 0  かつ  トレード数が統計的に十分
```

> **正直な期待値の置き方（不変）**：個人のアルゴ・スイングの大半もコストとギャップに負ける。
> 本設計は「エッジがあれば規律よく抽出し、なければ安く早く見切る」ためのもので、利益は保証
> しない。投入資金は全損しても生活に影響しない範囲に限定。本書は投資助言ではない。

## 1. 何が変わり、何が変わらないか

### 変わる
| 項目 | v2（日中） | v3（スイング） |
|------|-----------|----------------|
| 保有時間 | 当日クローズ | 数日〜最大2週間（`max_holding_days`） |
| 必要データ | 分足（有料/ライブ蓄積） | **日足（J-Quants 無料/有料で可）** |
| 主力指標 | VWAP・日中乖離 | 移動平均乖離（zスコア）／N日ブレイク |
| 執行 | 指値でスプレッド捕獲 | 翌営業日の寄り（or 指値）で建て |
| 最大の敵 | スプレッド・滑り | **持ち越し金利・オーバーナイトギャップ** |
| 検証可能性 | 分足が要る | **無料データで即バックテスト** |

### 変わらない（絶対原則）
- DRY_RUN ファースト（既定 True、人間が明示解除）
- 全発注は `risk_manager.py` を通す（サーキットブレーカー・回数上限・期待値ゲート）
- 期待値はコスト控除後でのみ評価（evaluator が cost_model を必須にして強制）
- 板情報・歩み値を主軸にしない（むしろスイングでは不要）
- バックテストのバイアス対策（§7）を必ず通す

## 2. 戦略（トラックA・両方を比較検証）

### 2-1. 日足スイング平均回帰（`strategy/swing_reversion.py`）
- エントリー：終値の `lookback` 日 z スコアが ±`entry_z` 超で逆張り
  （売られすぎ→ロング／買われすぎ→ショート）。
- イグジット：移動平均（`lookback`）への回帰で利確／ATR×`atr_stop_mult` で損切り／
  `max_holding_days` でタイムストップ。
- 容量制約ニッチ（中小型の行き過ぎ）と相性が良い。

### 2-2. 日足ブレイクアウト・モメンタム（`strategy/swing_momentum.py`）
- エントリー：終値が直近 `breakout_lookback` 日の高値上抜けでロング／安値下抜けでショート
  （前日までの窓の極値と比較＝未来非参照）。
- イグジット：固定目標なし。トレンドが続く限り保有し、ATR 損切り／タイムストップで出る。

### 2-3. 共通エンジン（`strategy/swing.py`）
両戦略はエントリーシグナルと目標系列だけを与え、保有・出口の状態機械を共有する：
- 約定は**翌営業日の寄り**で（同バー内の未来参照を避ける）。
- **ギャップを保守的にモデル化**：損切りは寄りが既に割っていれば寄り値（不利）で約定。
  目標利確は寄りで飛び越えても目標値どまり（楽観を避ける）。同バーで両方触れたら損切り優先。
- 単一ポジション制。出口の翌バー以降で次のエントリーを探す。

## 3. リスク管理（v2 §6 をほぼ踏襲＋スイング固有）

`config/settings.py: RiskParams` / `strategy/risk_manager.py`：
- 1トレード最大リスク 0.5%（`position_sizer` が損切り幅から株数を逆算）
- 1銘柄 20% / 同時保有 5 銘柄 / 1日損失上限 3%（サーキットブレーカー）/ 1日回数上限
- 期待値ゲート（直近の実効期待値が負なら新規建て停止）

**スイング固有の注意**：
- **オーバーナイトギャップ**で損切りが滑る（寄りで飛ぶ）。サイジングは保守的に。決算跨ぎの
  回避など、イベント前のフィルタを検討（将来）。
- **同時保有が日跨ぎで重なる**。dry_run/執行はイベント駆動で同時保有数と日次損益を管理。
- トレード数が日中より減るため、**統計的十分性は銘柄ユニバースの広さ**で稼ぐ。

## 4. データ（ここが最大の利点）

- **J-Quants 日足**（`data/fetcher.py`）：無料=過去2年/12週遅延、Light=5年、Standard=10年。
  **バックテストに遅延は無関係**。調整後（Adjustment*）価格で split バイアスを回避。
- **サバイバーシップ対策**：`get_listed_info(date_)` で過去日の上場ユニバース（後に廃止された
  銘柄を含む）を辿る。
- 分足・ライブ常時接続は**不要**。kabu はライブの発注時のみ（依然 Windows 必須）。
- 運用は **1日1回（引け後）に翌日の発注を決めるバッチ**で回せる（インフラ負荷が激減）。

> v2 の「純フォワード（ライブ実測の積み上げ）」は分足が買えない前提での回避策だった。
> v3 では日足が無料で手に入りバックテスト可能なので、**純フォワードは不要**。ただし
> `fill_monitor` による実約定率・実滑りの実測は、ライブ移行後の cost_model 較正に引き続き有用。

## 5. 検証フレームワーク（§7 のチェックリストは不変）

`backtest/evaluator.py`：
- コスト控除後の期待値 E[1トレード] > 0（**持ち越し金利を holding_days で計上**）
- ウォークフォワード（イン/アウトサンプル分離）で各区間安定
- 統計的に十分なトレード数 / 最大ドローダウン < 15%
- バイアス対策：ルックアヘッド（指標が causal）／サバイバーシップ（廃止銘柄込み）／
  滑り・約定率（cost_model＋将来 fill_monitor 実測）／マーケットインパクト／イン・アウト分離

**未対応として明示**：指値の不約定モデル化（現状は寄り/指値どおり約定の理想化）、
ギャップ時の損切り滑りは保守モデル済みだが実測較正が必要。

## 8. 既知のリスク（正直な開示・v3 追記）

- **持ち越し金利・貸株料**が保有日数ぶん効く（当日返済の前提が無い）。cost_model で必ず計上。
- **オーバーナイトギャップ**：決算・夜間の海外市場で寄りが飛び、損切りが想定を超え得る。
- スイング日足戦略は日中マイクロ構造より**競合が多い**面はある。容量制約ニッチ（中小型）で差別化。
- インフラ依存（kabuステーション常時起動はライブ発注時のみに軽減。ただし Windows 必須は不変）。
- 大半の個人アルゴはコストとギャップに負ける。良いバックテスト成績は楽観バイアスの塊（§7必須）。
```
